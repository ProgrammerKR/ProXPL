# ProXPL Security and Stability Fixes (v1.3.2+)

This document summarizes the resolution of critical issues, notable bugs, and key warnings across the ProXPL codebase to ensure runtime safety, stability, and proper project execution.

## Critical Fixes

### 1. Command Injection Guard
* **Location:** `prm_exec()` in `src/prm/commands/cmd_core.c`
* **Issue:** `argv[2]` was handed directly to `prm_exec()` with zero sanitization, leading to a potential command injection vulnerability if user inputs contained shell metacharacters.
* **Resolution:** Implemented input validation using the `isValidPrmArg()` function. Any command string containing shell metacharacters (`&`, `|`, `;`, `>`, `<`, `$`, `\`) is now rejected before execution.

### 2. VM Source File Lifecycle (Use-After-Free)
* **Location:** `src/proxpl_api.c`, `src/main.c`, and `src/runtime/vm.c`
* **Issue:** The source code string was `free()`d immediately after `interpret(pvm, source)` returned. However, the VM's token scanner stores raw `const char*` pointers into the source buffer, causing dangling pointers (use-after-free) within generated bytecode chunks.
* **Resolution:** Solved by moving `source` tracking into the VM lifecycle. A dynamic `sourceFiles` array was added to the `VM` struct (managed via `trackSource()`). The `freeVM()` function now safely frees these memory buffers upon teardown, allowing tokens inside generated chunks to safely retain references during execution.

### 3. Null Pointer Dereference Check
* **Location:** `src/main.c` (`repl()` function)
* **Issue:** `newFunction()` can return `NULL` on out-of-memory errors. The result was immediately passed into `OBJ_VAL()` and pushed onto the VM stack without any bounds checking, causing a null-dereference crash.
* **Resolution:** Implemented an explicit `NULL` check when allocating `newFunction()`. It will now safely log an out-of-memory error and continue, instead of crashing the stack pipeline.

## Bug and Stability Improvements

### 1. REPL Exit Unblocked
* **Location:** `src/main.c`
* **Issue:** The `exit` command check in the REPL ran *before* the newline was stripped, making it impossible to quit the REPL on Windows due to the lingering `\r` (CRLF).
* **Resolution:** Adjusted the order of `CRLF` stripping so that the REPL successfully strips the carriage return and newline characters before validating against the `"exit"` string.

### 2. Script Argument Swallowing
* **Location:** `src/main.c`
* **Issue:** The `.prox` file extension check was improperly placed within the `argc >= 3` branch, causing script arguments to be swallowed or incorrectly routed.
* **Resolution:** Re-ordered the script-argument CLI logic. `argv[1]` is now deterministically processed as a generic file to execute unless it matches a strictly defined CLI subcommand (like `run` or `build`).

### 3. Global Extern Declarations
* **Location:** `src/main.c`
* **Issue:** The global `vm` symbol was used in `main.c` but was never `extern`-declared, causing compilation failures in strict C99 environments.
* **Resolution:** Defined `extern VM vm;` explicitly in `main.c` to resolve static linking and scoping problems around the global virtual machine state.

### 4. Makefile Correctness
* **Location:** `src/Makefile`
* **Issue:** The `Makefile` was missing roughly half of the codebase in the `SOURCES` list, including the garbage collector, IR pipeline, optimizer, API layer, and the Package Manager (PRM), resulting in broken builds.
* **Resolution:** Synchronized the `Makefile` source mapping. The `SOURCES` block now correctly encompasses all 57 `.c` modules across all subdirectories necessary for a proper binary build.

## Refactoring and Warnings Addressed

### 1. `snprintf` String Allocation
* **Location:** `src/main.c`
* **Issue:** `snprintf` output truncation occurred when the UI app name was long due to a fixed-size buffer allocation.
* **Resolution:** Safely refactored the UI transpiler string builder logic. Directory paths (`dist_<app_name>`) are now dynamically allocated to match the exact string length before formatting via `sprintf()`, bypassing output truncation entirely.

### 2. Graceful Error Handling
* **Location:** `src/main.c`
* **Issue:** Error exit paths (`exit(65)`, `exit(70)`, `exit(74)`) lacked cleanup, leading to memory leaks on failures.
* **Resolution:** Overhauled all failure states inside the `main.c` pipeline blocks to guarantee a `freeVM(&vm)` call is executed before process termination to properly clean up VM resources.

### 3. DRY Code (`readFile`)
* **Location:** `src/main.c`, `src/proxpl_api.c`, `src/utils/file_utils.c`
* **Issue:** The `readFile()` helper function was duplicated in both `main.c` and `proxpl_api.c`, increasing the risk of code divergence.
* **Resolution:** Consolidated the local `readFile()` functionality into `src/utils/file_utils.c` and exposed it via `include/file_utils.h`. Both files now safely import and use the shared implementation.
