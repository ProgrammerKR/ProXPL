## ✨ New Features

### 1. Cross-Compilation
Build native binaries for any supported platform from any host.

```bash
# Build for Linux from Windows
proxpl build --target x86_64-linux app.prox

# Build for macOS ARM from Linux  
proxpl build --target aarch64-macos app.prox

# Build for Windows from macOS
proxpl build --target x86_64-windows app.prox

# Build for all platforms
proxpl build --target all app.prox
```

**Supported Target Triples:**
| Target | Architecture | OS |
|--------|-------------|-----|
| `x86_64-linux` | x64 | Linux |
| `x86_64-windows` | x64 | Windows |
| `x86_64-macos` | x64 | macOS |
| `aarch64-linux` | ARM64 | Linux |
| `aarch64-macos` | ARM64 | macOS (Apple Silicon) |
| `aarch64-windows` | ARM64 | Windows on ARM |
| `wasm32-wasi` | Wasm | WASI Runtime |
| `riscv64-linux` | RISC-V | Linux |

**Implementation:**
- LLVM cross-compilation via target triples
- Bundled sysroot headers for each target
- Static linking by default for portable binaries
- Binary size optimization via LTO (Link-Time Optimization)

### 2. Embedded Scripting API (`libproxpl`)
Embed ProXPL as a scripting engine in C/C++/Rust applications.

```c
// C Application embedding ProXPL
#include <proxpl.h>

int main() {
    ProxVM* vm = proxpl_new();
    
    // Register a native function
    proxpl_register(vm, "greet", my_greet_func);
    
    // Execute ProXPL code
    ProxResult result = proxpl_eval(vm, "greet(\"World\");");
    if (result.error) {
        printf("Error: %s\n", result.message);
    }
    
    // Execute a file
    proxpl_exec_file(vm, "scripts/game_logic.prox");
    
    // Call a ProXPL function from C
    ProxValue val = proxpl_call(vm, "calculate_score", 2,
        proxpl_int(100), proxpl_string("bonus"));
    
    printf("Score: %d\n", proxpl_as_int(val));
    
    proxpl_free(vm);
    return 0;
}
```

**API Surface:**
| Function | Description |
|----------|-------------|
| `proxpl_new()` | Create new VM instance |
| `proxpl_free()` | Destroy VM and release memory |
| `proxpl_eval()` | Execute a string of ProXPL code |
| `proxpl_exec_file()` | Execute a `.prox` file |
| `proxpl_register()` | Register native C function |
| `proxpl_call()` | Call a ProXPL function from C |
| `proxpl_get_global()` | Read a global variable |
| `proxpl_set_global()` | Set a global variable |
| `proxpl_sandbox()` | Enable sandboxed execution (restricted I/O) |
| `proxpl_set_memory_limit()` | Set memory ceiling for VM |
| `proxpl_set_timeout()` | Set execution timeout |

**Build:**
```bash
# Build as shared library
cmake -DBUILD_SHARED_LIB=ON ..
make

# Output: libproxpl.so / proxpl.dll / libproxpl.dylib
```

### 3. Security Hardening

**Sandboxed Execution:**
```javascript
// sandbox.prox — runs with restricted capabilities
@sandbox(
    allow_fs: false,
    allow_net: false,
    allow_exec: false,
    memory_limit: "256MB",
    timeout: "30s"
)
func untrusted_code() {
    // Cannot access filesystem, network, or system commands
    // Memory and time limited
    let result = heavy_computation();
    return result;
}
```

**Security Features:**
| Feature | Description |
|---------|-------------|
| **Capability-based security** | Code explicitly requests permissions |
| **Memory limits** | Per-sandbox memory ceilings |
| **Execution timeouts** | Prevent infinite loops |
| **Filesystem sandboxing** | Restrict to specific directories |
| **Network restrictions** | Allowlist-based network access |
| **Audit logging** | Record all privileged operations |
| **Stack overflow protection** | Configurable stack depth limits |
| **Integer overflow detection** | Runtime checks in debug mode |

### 5. API Freeze & Stability Guarantees
Formal stability policy for the v1.x series.

**Stability Tiers:**
| Tier | Meaning | Guarantee |
|------|---------|-----------|
| **Stable** | Core language syntax, `std.core`, `std.io` | No breaking changes until v2.0 |
| **Beta** | `std.net`, `std.db`, actors, macros | Minor changes possible with deprecation warnings |
| **Experimental** | Quantum, AI-Native, Hardware-Accelerated | May change significantly |

**Deprecation Policy:**
1. Feature marked `@deprecated` with migration guide
2. Compiler warning for 2 minor releases
3. Removal only in next major version (v2.0)
