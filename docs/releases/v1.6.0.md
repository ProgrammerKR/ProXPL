# ProXPL v1.6.0: The "Studio Launch" Release

**Target Release Date:** Q1 2027  
**Status:** Planned  
**Codename:** Studio Launch

---

## 🌟 Overview

ProXPL v1.6.0 delivers the **ProX Studio Alpha** — a dedicated IDE experience — alongside the official **PRM Registry** for centralized package distribution, a complete **Testing Framework** (`std.test`), and **Macro/Metaprogramming** capabilities.

---

## ✨ New Features

### 1. ProX Studio (Alpha)
A lightweight, dedicated IDE for ProXPL built on Electron/Tauri.

**Features:**
- Syntax highlighting with semantic tokens
- Integrated terminal with ProXPL REPL
- Project manager with `project.pxcf` awareness
- Integrated debugger (built on v1.5.0 DAP)
- PRM package browser and installer
- One-click build & run
- Dark/Light themes with the ProXPL design language
- Built-in documentation viewer

**Technical Stack:**
- Backend: Tauri (Rust) for native performance
- Frontend: ProXPL-powered UI framework (dogfooding)
- LSP integration for IntelliSense
- Embedded terminal emulator

### 2. PRM Registry (Official Package Registry)
Centralized package distribution and discovery.

```bash
# Publish your package
prm login
prm publish

# Install from registry
prm install json-parser
prm install http-server@2.1.0

# Search packages
prm search "web framework"
```

**Infrastructure:**
| Component | Technology |
|-----------|-----------|
| **API Server** | ProXPL + Go hybrid backend |
| **Package Storage** | S3-compatible object store |
| **Database** | PostgreSQL (package metadata) |
| **CDN** | CloudFlare for global distribution |
| **Website** | registry.proxpl.dev |

**Features:**
- Semantic versioning enforcement
- Dependency resolution with SAT solver
- Package signing and verification
- Download statistics and popularity scoring
- Organization/team package ownership
- Audit logs and vulnerability advisories

### 3. Testing Framework (`std.test`)
Production-ready testing without external dependencies.

```javascript
use std.test;

describe("Math operations") {
    test("addition") {
        assert_eq(1 + 1, 2);
        assert_eq(0.1 + 0.2, 0.3, tolerance: 0.001);
    }

    test("division by zero") {
        assert_throws(func() { 1 / 0; });
    }

    test("fibonacci correctness") {
        assert_eq(fibonacci(10), 55);
        assert_eq(fibonacci(0), 0);
    }

    bench("fibonacci performance", iterations: 1000) {
        fibonacci(30);
    }
}
```

**Features:**
| Feature | Description |
|---------|-------------|
| `test` blocks | Define individual test cases |
| `describe` blocks | Group related tests |
| `assert_eq` | Equality assertions with diff output |
| `assert_ne` | Inequality assertions |
| `assert_true/false` | Boolean assertions |
| `assert_throws` | Exception expectation |
| `assert_type` | Type checking assertions |
| `bench` blocks | Built-in micro-benchmarking |
| `setup` / `teardown` | Test lifecycle hooks |
| `skip` | Skip individual tests conditionally |

**CLI Integration:**
```bash
prm test                    # Run all tests
prm test --filter "math"    # Run matching tests
prm test --parallel         # Run tests in parallel
prm test --coverage         # Generate coverage report
prm test --bench            # Run benchmarks only
```

### 4. Macros & Metaprogramming
Compile-time code generation.

```javascript
// Simple declarative macros
macro log!(level, msg) {
    if (LOG_LEVEL >= level) {
        print("[" + to_string(level) + "] " + msg);
    }
}

log!(DEBUG, "Starting process");

// Derive macros for automatic trait implementation
@derive(Printable, Comparable)
class User {
    let name: string;
    let age: int;
}

// Compile-time assertions
static_assert(sizeof(int) == 8, "Expected 64-bit integers");
```

**Implementation Details:**
- Hygienic macro expansion (no name collisions)
- `macro` keyword for declarative macros
- `@derive` decorator for automatic trait implementations
- `static_assert` for compile-time validation
- AST-level transformations before type checking

```bash
# Output: flamegraph.svg
```
